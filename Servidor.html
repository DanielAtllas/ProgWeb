Prog. Web - Servidor

Dando continuidade aos estudos de programação web, de forma a criar aplicações que possuam uma interação direta com a rede de internet e possa ser disponibilizada aos usuários, iremos partir agora para a configuração do servidor da aplicação. Este servidor, tem como função ser a "casa" de abastecimento do programa, onde este ficará armazenado todo tipo de dado recorrente ao programa em questão, como dados, scripts e etc.

Configurando o Ambiente de Servidor Web

Por estarmos utilizando da IDE do eclipse, iremos utilizar do TomCat como nosso servidor para este serviço. O tomcat, funciona como uma base de dados criada a partir da própria máquina que o está utilizando, ou seja, este fará da máquina do programador como um servidor privado e a partir dele, o programa poderá ser exibido para toda a rede.

Diante disso, partiremos para o processo de configuração do TomCat para com o eclipse. Para isso, seguiremos uma pequena série de passos, onde:

  1. No diretório principal de sua máquina (c:/), crie uma pasta de nome TomCat (Esta será para um redirecionamento mais simples de encontro do TomCat, uma vez que será preciso acioná-lo posteriormente).

  2. Será preciso agora baixar o arquivo .zip no próprio site do fornecedor: tomcat.apache.org, escolhermos a versão que será utilizada e simplismente baixa-la na pasta criada anteriormente.

  3. Após isso, basta extrair o arquivo na pasta em questão, ou seja, o TomCat é uma aplicação direta e por isso, não é preciso executa-lo ou instala-lo na máquina.

Tendo o TomCat extraido na pasta desejada, este já estará disponível para uso nas aplicações web que forem criadas pelo eclipse, deviendo agora apenas configura-lo para acesso. Então, ao abrir o eclipse, ao invés de criarmos um projeto como java project, iremos escolher a opção de Dynamic Web Projct. Você verá que a janela de configuração inicial do projeto será um pouco maior que a padrão.

Nisso, teremos de ir na opção Target Runtime e New Runtime. Então uma pequena janela de opções será aberta e será necessário escolher a de TomCat, que está dentro da pasta Apache (Você verá que muitas opções de TomCat em diferentes versões serão exibidas, cabendo ao programador escolher a que melhor te atende - Recomenda-se sempre escolher a versão mais recente).

Feito isso, iremos na opção next e agora na opção Browse para definirmos o local em que se encontra o TomCat extraido, ou seja, iremos ir de encontro no diretório em que está o tomcat, selecionar sua pasta extraida e pronto. Bssta agora apertar em finish e encerrar as configurações do projeto para inicia-lo.

OBS: Caso todos os passos tenham sido corretamente seguidos, na parte inferior do Eclipse (após o projeto estar criado) estará a opção de Servers como na imagem abaixo:

Perceba a indicação de que nenhum servidor está configurado até então, descrito em azul. Clicaremos nesta opção e faremos o processo semelhante a anteriormente, escolhendo a versão do TomCat que será usada e clicando em finish. Feito isso, teremos todo o ambiente de aplicação web configurado e pronto para uso.

OBS: Após a conclusão de todas as etapas anteriores, será possível identificar um projeto criado instantaneamente no canto esquerdo do eclipse, de nome Servers, este projeto não deverá ser nunca excluido e também não poderá ser fechado, pois para que a aplicação funcione com o Tomcat, este projeto deverá estar aberto para uso no eclipse.

Criando um Projeto Web

Bom, como foi dito anteriormente, o processo de criação de um projeto web é um pouco diferente de um projeto comum, portanto, é perceptivel que as opções de bibliotecas e diretórios do projto são um pouco mais extensas e com mais opções. Basicamente, nossa interação se resumirá à pasta de nome src, onde iremos inserir as classes .java na pasta de nome java e as classes páginas html na pasta de nome webapp.

OBS: Na pasta de nome WEB-INF estará presente uma segunda pasta de nome lib. Nesta, iremos armazenar todas as bibliotecas que usaremos no programa, sendo elas:

  • eclipselink
  • jakartas.persistence
  • jakarta.servlet.jsp.jstl
  • jakarta.servlet.jsp.jstl-api
  • mysql-connector-java
